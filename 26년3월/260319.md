- let / const

  - 변수 스코프의 가장 위쪽에서 변수의 초기화가 완료될때까지 TDZ라는 공간에 존재

  - 즉, 호이스팅이 발생하지 않는게 아님

  - Temporal Dead Zone

    - 변수를 사용하는 것을 허용하지 않는 공간

    - CF) Var의 경우엔 변수 생성 이후 바로 메모리 공간에 변수의 공간을 할당

      - (값을 할당하는 것은 아님 / 따라서 Undefined로 나옴)

      ````
      console.log(num1);
      var num1 = 123;
      ````

    - let과 const는 변수의 값이 할당되기 전까지 메모리 공간을 할당 X

- 함수 표현식

  - 호이스팅 적용되어 TDZ에 들어감

  ```
  hoisted();
  
  const hoisted = () => {
  	console.log('hoisted');
  } // uncaught ReferenceError
  ```

  

- 객체 생성

  - 생성자 방식 또는 리터럴 방식으로 생성 가능

  - 생성자 방식

    - ```
      let obj = new Object();
      console.log(obj); // {} 출력
      ```

  - 리터럴 방식

    - ```
      let obj = {};
      console.log(obj);
      ```

  - value에는 문자열 , undefined, 함수 등 어떤 자료형을 넣어도  오류 발생 X

    - ```
      let book = {
      	title : "어쩌구",
      	author: "kdw",
      	cateogry : '문학',
      	year : undefined,
      	color : function () {
      		console.log('oragne');
      	},
      };
      ```

  - 객체에서 property를 뽑아낼때는 점 표기법('.')으로 다음과 같이 사용가능하지만 배열처럼 괄호표기법도 가능

    - ```
      let car = {
          name: '붕붕',
          model: 'morning',
          color: 'black',
      };
      
      console.log(car.name);
      console.log(car.model);
      console.log(car.color);
      
      
      let car = {
          name: '붕붕',
          model: 'morning',
          color: 'black',
      };
      
      console.log(car["name"]);
      console.log(car["model"]);
      console.log(car["color"]);
      ```

      - 통상 점 표기법을 많이 사용하지만, 다음과 같이 함수 매개변수가 객체의 key값으로 활용되거나 하는 방식에선 사용가능

    - ```
      let car = {
          name: '붕붕',
          model: 'morning',
          color: 'black',
      };
      
      const getValue = (key) => {
          console.log(car[key]);
      };
      
      getValue('name');
      ```

  - 프로퍼티 수정

    - ```
      const cat = {
      		name: '야옹이',
          age: 2,
      	  color: 'white'
      };
      
      cat.age = 1;
      cat['color'] = 'yellow';
      ```

    - const로 선언한 객체의 프로퍼티 값 역시 수정 가능

    - why?

      - 객체 프로퍼티의 값을 수정하는 것과 객체 'cat'을 수정하는 것은 다르기 때문

        - 객체 생성시 고유한 id값을 지님
        - cat의 내부 프로퍼티를 변경하는 것은 cat의 고유한 id값을 변경하는 것과 다름
        - 다음과 같은 경우는 에러 발생 가능

      - ```
        const cat = {
        		name: '야옹이',
            age: 2,
        	  color: 'white'
        };
        
        cat = {
        		age: 3,
        }
        ```

  - 프로퍼티 삭제

    - delete 키워드 사용

      - ```
        const cat = {
        	name : '어쩌구',
        	age : 1,
        	color : 'white'
        }
        
        delete cat.color;
        delete cat['age'];
        ```

        - color와 age 프로퍼티 삭제 완료

  - 객체 메서드 생성시에는 화살표 함수보다 함수선언식으로 선언하는게 낫다? 

    - 화살표 함수의 경우 this 키워드 사용시 -> window객체만 접근 가능 
    - But, 함수 선언식의 경우 this키워드 사용시 객체 내부의 프로퍼티에 접근 가능

- 배열

  - 비원시타입 / 객체와 동일하게 생성자함수 또는 리터럴을 사용해서 생성가능

  - 생성자 방식

    - ```
      let arr = new Array();
      console.log(arr); // []
      
      let arr1 = new Array(1,2,3);
      let arr2 = new Array(3);
      
      console.log(arr1); // [1,2,3];
      console.log(arr2); // [비어있음]
      ```

      - arr2는 3이라는 값이 **배열의 요소에 할당되는 것이 아니라 undefined라는 3개의 값이 배열에 할당되어 크기가 3인 배열이 생성**

    - 배열 생성자에 값을 할당할 때, 특정 값을 요소에 할당하기 위해서는 생성자 함수의 괄호 안에 여러 개의 값을 넣고, 원하는 크기의 공간을 특정 배열에 할당하기 위해서는 원하는 공간의 크기를 괄호 안에 작성

  - 리터럴 사용방식(더 보편적)

    - ```
      let arr = [];
      console.log(arr);
      
      
      let arr1 = [1, 2, 3];
      let arr2 = [3];
      
      console.log(arr1); // [1,2,3]
      console.log(arr2); // [3]
      ```

      - 생성자 방식과 다르게 더 명확함

  - 배열 요소
    - 자료형에 상관없이 작성 가능

  - 요소 추가

    - push
      - 마지막 인덱스에 요소 추가
    - unshift
      - 첫번째 인덱스에 요소 추가

  - 요소 수정

    - ```
      let animal = ['cat', 'dog', 'hamster'];
      animal = ['cat', 'rabbit', 'hamster'];
      animal[2] = 'parrot';
      
      console.log(animal); // ['cat', 'rabbit', 'parrot']
      
      
      ---
      
      const animal = ['cat', 'dog', 'hamster'];
      animal = ['cat', 'rabbit', 'hamster'];
      
      console.log(animal); // TypeError
      
      ---
      
      const animal = ['cat', 'dog', 'hamster'];
      animal[2] = 'parrot';
      
      console.log(animal); // ['cat', 'dog', 'parrot']
      ```

      - const로 선언되었어도 배열의 요소 자체는 수정 가능하나 const로 선언한 animal자체는 수정불가

  - 요소 삭제

    - pop()
      - 마지막 요소 삭제
    - shift()
      - 첫번째 요소 삭제
    - splice(a,b)
      - a번째 인덱스부터(포함하여) b개의 요소를 삭제
      - 특정 인덱스의 요소 삭제

- for..of

  - 배열에서 주로 사용

  - ```
    let arr = [1,2,3,4,5];
    for(let i of arr){
    	console.log(i); // 1,2,3,4,5
    }
    ```

- for..in

  - 객체에서 주로 사용

  - 반복문에서 사용되는 변수는 객체의 'key값'

  - ```
    let person = {
        name: "홍길동",
        age: 25,
        height: 180,
    };
    
    for (let key in person) {
        console.log(`${key} : ${person[key]}`);	// 
    }
    ```
