### Filter / OnceperRequestFilter

- Filter

  - Servlet Spec

- OnceperRequestFilter

  - Spring Spec(Spring Web 모듈 내 존재)

  - 1회만 딱 실행됨

    - 'a single execution per request dispatch'

    - 내부적으로 request attribute를 사용해 이미 사용된 필터인지를 표시해서 중복실행을 방지

      - 다음과 같이 getAlreadyFilteredAttributeName()을 통해서 필터가 다시 동작하지 않게끔 설계

      - ```java
        String attrName = getAlreadyFilteredAttributeName();
        
        if (request.getAttribute(attrName) != null) {
            filterChain.doFilter(request, response);
            return;
        }
        
        request.setAttribute(attrName, Boolean.TRUE);
        try {
            doFilterInternal((HttpServletRequest) request, (HttpServletResponse) response, filterChain);
        } finally {
            // 내부 정책에 따라 정리
        }
        ```

        

  - 그런 의미로 보안 / 토큰 인증/ 등에서는 한 요청에서 딱 1회만 실행되도록 수행

  - 따라서 Spring Security에서 해당 Filter를 상속하는 방식으로 동작

  **“단순히 가로채고 싶으면 Filter, 인증처럼 한 번만 안정적으로 실행돼야 하면 OncePerRequestFilter.”**
