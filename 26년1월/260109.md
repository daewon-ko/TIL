### SPA / CSR, SSR 

- SPA
  - 페이지 전체 리로드 없이 URL과 화면이 바뀌는 구조
    - react나 vue가 SPA방식이라고 할 수 있는데 말 그대로 Page(HTML) 자체가 한 개인셈
    - 페이지 이동시에는 JS코드로 dom객체 / Component 등을 제어해서 화면을 새롭게 그려준다.
    - 따라서 새로고침 없이 부드럽게 화면이 이동가능하며 상태관리에서도 훨씬 이점이 높음
- CSR이 SPA를 사용하는 보편 방식이나 반드시 그러한 것은 아님
  - SSR + SPA도 아래 구조로 구성 가능
    - next.js / nuxt / sveltkit
    - 이렇게 사용하는 이유는 SPA의 장점도 취하면서 SSR을 사용해서 SEO도 취하기 위함이다.
- 전통적인 SSR은 모바일웹만 만들었음
  - ex) Spring + jsp / SpringBoot + thymleaf
  - 그러나 현재는 반드시 그러하지 않음
    - 위에서 말한것처럼 SSR이라고 해도 Next.js / Nuxt / SveltKit를 이용해서 SSR이되 SPA로 구성해서 웹앱처럼도 구성가능함.
  - SSR은 말 그대로 HTML을 서버에서 생성하느냐 여부에 따라 결정된다.
    - SSR
      - HTML을 서버가 만들어주고 브라우저는 js로 그 HTML을 그려주기만 한다.
      - JS는 이벤트 처리 /약간의 동적 효과를 담당한다.
      - 화면 제어의 주체가 '서버'이다.
    - CSR
      - 서버는 HTML의 뼈대만 제공한다.
      - JS가 화면을 만든다. 
      - 즉 화면 제어의 주체가 브라우저 JS이다.
- JS 번들은 CSR이든 SSR이든 모두 다 브라우저에게 제공된다.
